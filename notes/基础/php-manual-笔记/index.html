Apache PHP 运行模式

CGI(Common Gateway Interface) / Fast_CGI (常驻 long-live 型的 CGI)

phpinfo() => Server API CGI/FastCGI

php54之前，php-fpm(第三方编译)是管理器，php-cgi是解释器。

php54之后，php-fpm(官方自带)，master 与 pool 模式。php-fpm 和 php-cgi 没有关系了。php-fpm又是解释器，又是管理器。

php NTS 即None-Thread Safe，一般以fast_cgi方式运行的时候选择这个版本。

Apache模块(mod_php5模块的形式集成)

phpinfo() => Server API Apache 2.0 Handler

以apache模块运行PHP时，How to change configuration settings

模式 含义
PHP_INI_USER 可在用户脚本（例如 ini_set()）或 Windows 注册表（自PHP 5.3 起）以及 .user.ini 中设定
PHP_INI_PERDIR 可在 php.ini，.htaccess 或 httpd.conf 中设定
PHP_INI_SYSTEM 可在 php.ini 或 httpd.conf 中设定
PHP_INI_ALL 可在任何地方设定

配置运行目录

php.ini 的 doc_root ，或设置环境变量 PHP_DOCUMENT_ROOT 来定义 PHP 脚本主目录。

doc_root = "C:/Tools/phpStudy/PHPTutorial/WWW"

也能通过 apache.conf 设置（能被doc_root覆盖）

DocumentRoot  "C:/Tools/phpStudy/PHPTutorial/WWW"

报错调试

  • 利用 error_reporting() 来帮助使代码更安全并发现变量使用的危险之处。

    // 报告所有 PHP 错误 四式
error_reporting(E_ALL);
error_reporting(-1);   // 0为关闭报错
ini_set('error_reporting', E_ALL);
ini_set('display_errors', 'On');

  • 把 php.ini 中的 display_errors 设为 off 来关闭所有的错误显示，但依然可以ini_set('display_errors', 'On'); 在php脚本临时打开。

  • 打开 ini 文件内的 log_errors 选项，并通过 error_log 指定用于记录错误信息的文件。

魔术引号

magic_quotes_gpc 自 PHP 5.3.0 起废弃并将自 PHP 5.4.0 起移除。

自动将进入 PHP 脚本的数据进行转义。当选项打开时，所有的 '（单引号），"（双引号），\（反斜线）和 NULL 字符都会被自动加上一个反斜线进行转义。这和 addslashes() 作用完全相同。
