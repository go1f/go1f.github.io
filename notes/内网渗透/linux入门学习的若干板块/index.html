操作系统

进程管理、储存管理、设备管理、文件管理、作业管理

用户与用户组

相关文件

/etc/passwd

  • -rw- r– r–

  • 用户信息

  • 1.用户名 2.密码 3.UID 4.GID组号 5. 备注、注释、描述信息 6.用户主目录 7.shell

/etc/shadow

  • -r– — —

  • 密码存储

  • $5表示SHA256、$6 SHA512、$1 MD5

  • 清空密码 passwd -d 用户名

  • *或！表示用户锁定

/etc/group

  • 最后一位表示添加到该组的用户

添加账号

  • useradd 新用户名 直接创建，没有设置密码、home目录
  • adduser 新用户名 创建并交互式设置密码，用户信息等

用户管理

  • id 显示用户 uid、gid、所属组等信息

  • 切换用户 su <user>

  • 修改密码 passwd <user>

  • 删除用户 userdel <user> -r 删除家目录

  • usermod <user>

    -L 锁定用户，禁止密码登录，但root依然可以切换到用户 -U 解锁用户 -g 修改到指定群组 -G 附加到群组 -l 改账号名 -u 修改UID（若vi修改为0，伪装root用户） -d 修改家目录 -s 修改shell

    常见shell

    • /bin/bash 允许登录
    • /bin/sh 不带用户名，没有高亮等特色
    • /bin/false 不能登录，没有回显
    • /usr/sbin/nologin 不能登录，提示账号不能登陆

文件权限系统

对文件

r （read）：可读取

w （write）：可以编辑、新增或者是修改该文件的内容（但不含删除该文件）；

x （eXecute）：可以被系统执行

对目录

  • r （read contents in directory）

查询该目录下的文件名数据，即ls。若没有可读权限，但已知文件路径，一样可以找到文件。

  • w （modify contents of directory）

创建新的文件与目录； 删除已经存在的文件与目录（不论该文件的权限为何！） 将已存在的文件或目录进行更名； 搬移该目录内的文件、目录位置。 总之，目录的w权限就与该目录下面的文件名异动有关就对了啦！

  • x （access directory）

设为 工作目录（work directory），即cd

读一个文件时，要具有“这个文件所在目录的 x 权限”

内容 r w x
文件 详细数据data 读到文件内容 修改文件内容 执行文件内容
目录 文件名 读到文件名 修改文件名 进入该目录的权限（key）

ls -[a][l] a.txt ls -ld mydir 文件夹 d表示dir -a 显示.开头的隐藏文件

修改文件属性

改变分组 chgrp

chgrp [-R] <user> <dirname/filename> -R 表示递归，同次目录下的所有文件、目录都更新

改变文件持有者 chown

chown [-R] 帐号名称 文件或目录 chown [-R] 帐号名称:群组名称 文件或目录

改变权限 chmod

  • 数字类型改变文件权限

chmod [-R] xyz 文件或目录 xyz : 就是刚刚提到的数字类型的权限属性，为 rwx 属性数值的相加。

  • 符号类型改变文件权限
| chmod | u g o a | +（加入）-（除去） =（设置） | r w x | 文件或目录 |
例：
[root@study ~]# chmod  u=rwx,go=rx  abc.txt
[root@study ~]# ls -al abc.txt
-rwxr-xr-x. 1 root root 176 Dec 29  2013 abc.txt

更改文件时间(atime、mtime)

touch -t <201606080800> <filename>

stat <filename> #可查看文件时间详情 find . -type f -mtime -10 #查找10天内被修改的文件

ctime(change time) atime(Access time) mtime(modify time)

目录结构

/etc 系统中的配置文件，账户密码文件、各种服务的配置文件

/bin, /sbin, /usr/bin, /usr/sbin 系统预设的执行文件的放置目录，比如 ls 就是在/bin/ls 目录下的。

值得提出的是，/bin, /usr/bin 是给系统用户使用的指令（除root外的通用户），而/sbin, /usr/sbin 则是给root使用的指令。

/usr 系统软件资源目录 Unix Software Resource /usr/share /usr/include(C语言编译头文件) /usr/bin(非系统自检必需的可执行文件) /usr/sbin /usr/lib /usr/local(源码安装)

/var 常态可变动文件目录，存放着在不断扩充着的东西，我们习惯将那些经常被修改的目录放在这个目录下。包括各种日志文件。

/tmp 存放一些临时文件。

/proc 虚拟的目录，它是系统内存的映射，可以通过直接访问这个目录来获取系统信息。

/lib 系统最基本的动态连接共享库

/dev 存放的是Linux的设备与接口设备，/dev/null，/dev/zero，/dev/tty等。

/media /mnt 临时挂载目录

/boot 引导程序，内核等存放的目录
